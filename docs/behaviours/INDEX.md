# behaviours Index

## Purpose

Product behavior specifications. `platform.md` is the source of truth for
what the product should do. Summary documents (`current-state.md`,
`e2e-checklist.md`) must stay derivable from it.

## File conventions

- Markdown files describing product behavior scenarios.
- `platform.md` is always the canonical spec — update it first when behavior
  changes.
- Summary docs are validated by `scripts/check-behaviour-docs.py`.
- Do not add implementation details here; those belong in `docs/ARCHITECTURE.md`.

- [platform.md](platform.md) — Canonical product behavior spec
- [current-state.md](current-state.md) — What's currently implemented
- [e2e-checklist.md](e2e-checklist.md) — Test coverage checklist
